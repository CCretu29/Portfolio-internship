Ticket system website

Project Description

This project was called "Passion project" as a final assignment for my semester two. The project was to create a ticket system website where users can create an account, log in, and buy a ticket for an event. The website was created using Node.js for back end, HTML and CSS for the front end and MongoDB as database.

Technologies Used

  • HTML
  • CSS
  • JavaScript
  • Node.js
  • MongoDB
  • Postman

Project Process

In this section, I will explain the steps I took to complete the project. This includes the initial planning, design, development, testing, and deployment phases. The first thing was to create the concept for this project. "Passion project" stands for a small project that shows the student passions. I chose to design the website and the create in back end as well, because bot design and programming are my passion. The theme of the website also stands as one of my passions, rock music. Around the same time as I was working on this project, AC/DC, a rock band that I love had a concert I was going to attend, and that is how I decided to make a ticket system for them. The project had all the steps needed starting with creating a project plan, answering research questions, and making a timeline. After I completed those steps, I did research about my competition, witch was other websites that sell tickets to rok concerts and I created a wireframe and test it on multiple users. After that, based on the user tests results, I created a high fidelity prototype and test it as well. I got a lot of useful feedback witch I used to improve my prototype. After my design was validated I started creating the website. I started with the back end, creating the server and the database. After that I created the front end using HTML, CSS, and JavaScript. I test the functionality in Postman before creating the front end. After the website was created I tested it on multiple users and made some changes based on the feedback I received. After that, I made a project report.
